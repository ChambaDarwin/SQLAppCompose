Dominando el JOIN
en MySQL

Los datos reales viven en varias tablas relacionadas entre sí. JOIN es el comando que las une para que puedas consultarlas juntas en una sola consulta.

¿Qué es JOIN?

JOIN combina filas de dos o más tablas basándose en una columna que tienen en común. En lugar de repetir todos los datos en una sola tabla gigante, las bases de datos relacionales dividen la información en tablas más pequeñas y las conectan con JOIN cuando las necesitas juntas.

🧩
Analogía del rompecabezas Imagina que tenés dos piezas de rompecabezas: una con el nombre y ciudad del cliente, otra con sus pedidos. JOIN encuentra la pieza encajante — el id del cliente — y las une para mostrarte el cuadro completo: quién compró qué. Sin JOIN, solo verías la mitad del rompecabezas.

Existen varios tipos de JOIN. Los más usados son:

INNER JOIN
Solo filas con coincidencia en ambas tablas
LEFT JOIN
Todas las filas de la izquierda, coincidan o no
RIGHT JOIN
Todas las filas de la derecha, coincidan o no
CROSS JOIN
Cada fila con cada fila (producto cartesiano)

Claves primarias y foráneas

Antes de usar JOIN, hay que entender las dos columnas que conectan las tablas.

PRIMARY KEY
Identificador único de cada fila. En la tabla clientes, el campo id es la PK. No puede repetirse ni ser NULL.
FOREIGN KEY
Referencia a la PK de otra tabla. En pedidos, el campo cliente_id apunta al id de la tabla clientes. Crea la relación entre tablas.
ON
La condición de unión. Le dice a MySQL qué columnas deben coincidir: ON clientes.id = pedidos.cliente_id
🔑
Analogía del número de empleado En una empresa, cada empleado tiene un número único (PK). En la tabla de nómina, ese número aparece en cada registro de pago (FK). JOIN es como buscar en la nómina el número de empleado y traer su nombre de la tabla de personal — conecta los datos sin repetirlos.

INNER JOIN — Solo coincidencias

INNER JOIN devuelve únicamente las filas que tienen coincidencia en ambas tablas. Si un cliente no tiene pedidos, no aparece. Si un pedido no tiene cliente válido, tampoco aparece.

🎯
Analogía de la intersección INNER JOIN es como la zona donde dos círculos se superponen en un diagrama de Venn: solo lo que está en ambos al mismo tiempo. Todo lo que está solo en un círculo queda fuera.
Clientes con sus pedidos (INNER JOIN)
SELECT clientes.nombre, clientes.ciudad, pedidos.producto, pedidos.monto FROM clientes INNER JOIN pedidos ON clientes.id = pedidos.cliente_id;
Resultado → Pedro Vega no aparece porque no tiene pedidos

LEFT JOIN — Todos los de la izquierda

LEFT JOIN devuelve todas las filas de la tabla izquierda (la que está antes del JOIN), aunque no tengan coincidencia en la tabla derecha. Donde no hay coincidencia, aparece NULL.

📋
Analogía de la lista de asistencia Es como la lista completa de estudiantes matriculados. Aunque algunos no asistieron a ninguna clase, siguen apareciendo en la lista — con un espacio vacío (NULL) donde deberían ir sus notas. LEFT JOIN garantiza que todos los de la lista izquierda aparezcan.
Todos los clientes, hayan pedido o no (LEFT JOIN)
SELECT c.nombre, c.ciudad, p.producto, p.monto FROM clientes c LEFT JOIN pedidos p ON c.id = p.cliente_id;

RIGHT JOIN — Todos los de la derecha

RIGHT JOIN es el espejo de LEFT JOIN: devuelve todas las filas de la tabla derecha, aunque no tengan coincidencia en la izquierda. En la práctica es poco usado — casi siempre se puede reescribir como LEFT JOIN cambiando el orden de las tablas.

LEFT JOIN equivalente
SELECT * FROM clientes c LEFT JOIN pedidos p ON c.id = p.cliente_id; -- Todos los clientes
RIGHT JOIN equivalente
SELECT * FROM pedidos p RIGHT JOIN clientes c ON c.id = p.cliente_id; -- Mismo resultado, tablas -- en orden invertido
💡
Preferí LEFT JOIN sobre RIGHT JOIN Son equivalentes en potencia, pero LEFT JOIN es más fácil de leer. La mayoría de desarrolladores y equipos usan LEFT JOIN consistentemente y cuando necesitan la otra tabla completa, la ponen primero en el FROM.

Alias de tablas

Cuando trabajas con varias tablas, escribir el nombre completo cada vez es tedioso. Los alias le dan un nombre corto a cada tabla para que el código sea más limpio.

Sin alias — verboso
SELECT clientes.nombre, clientes.ciudad, pedidos.producto, pedidos.monto FROM clientes INNER JOIN pedidos ON clientes.id = pedidos.cliente_id;
Con alias — limpio
SELECT c.nombre, c.ciudad, p.producto, p.monto FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id;
💡
Convención de alias Usa la primera letra de la tabla: clientes → c, pedidos → p. Si hay ambigüedad, usa las dos primeras: cl, pe. El alias se define después del nombre de la tabla en el FROM.

Múltiples JOINs

Podés encadenar tantos JOINs como necesites para unir más de dos tablas. Cada JOIN agrega una tabla nueva al resultado.

Supongamos que tenemos una tercera tabla productos:

Unir 3 tablas: clientes + pedidos + productos
SELECT c.nombre AS cliente, p.fecha AS fecha_pedido, pr.nombre AS producto, pr.categoria AS categoria, p.monto FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id INNER JOIN productos pr ON p.producto_id = pr.id ORDER BY c.nombre, p.fecha;
📌
Cada JOIN agrega una tabla No hay límite de JOINs en una consulta. Sin embargo, demasiados JOINs pueden volverla lenta. Para consultas complejas, asegurate de que las columnas usadas en ON tengan índices.

JOIN + WHERE y funciones

Podés combinar JOIN con WHERE para filtrar, y con GROUP BY + funciones de agregado para resumir datos de múltiples tablas.

Total comprado por cada cliente de Quito
SELECT c.nombre, COUNT(p.id) AS num_pedidos, SUM(p.monto) AS total_comprado FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id WHERE c.ciudad = 'Quito' GROUP BY c.id, c.nombre ORDER BY total_comprado DESC;

COALESCE para reemplazar NULL por 0 COALESCE(SUM(monto), 0) devuelve el SUM si existe, o 0 si es NULL. Muy útil con LEFT JOIN donde algunos grupos no tienen datos en la tabla derecha.

Errores comunes

1. Olvidar el ON — producto cartesiano accidental

✗ Sin ON (peligroso)
SELECT * FROM clientes INNER JOIN pedidos; -- Devuelve 4×5 = 20 filas -- combinando todo con todo -- CROSS JOIN implícito
✓ Con ON (correcto)
SELECT * FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id; -- Devuelve solo filas -- con coincidencia real ✓

2. Columna ambigua sin prefijo de tabla

✗ Ambiguo
SELECT nombre, id, monto FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id; -- "id" existe en ambas -- tablas → ERROR
✓ Con prefijo de tabla
SELECT c.nombre, c.id, p.monto FROM clientes c INNER JOIN pedidos p ON c.id = p.cliente_id; -- Prefijo c. y p. para -- identificar cada col ✓

3. Confundir LEFT JOIN con INNER JOIN

⚠️
¿Desaparecen filas en tu resultado? Si esperabas ver todos los registros de una tabla y algunos no aparecen, probablemente estás usando INNER JOIN cuando necesitas LEFT JOIN. INNER JOIN descarta silenciosamente las filas sin coincidencia — cambialo a LEFT JOIN y aparecerán con NULL donde no hay datos.

Resumen: JOIN completo

Reporte completo: todos los clientes con sus compras
-- Todos los clientes, con o sin pedidos, -- total comprado, ordenados de mayor a menor SELECT c.nombre AS cliente, c.ciudad, COUNT(p.id) AS num_pedidos, COALESCE(SUM(p.monto), 0) AS total_comprado, COALESCE(MAX(p.monto), 0) AS compra_mayor FROM clientes c LEFT JOIN pedidos p ON c.id = p.cliente_id GROUP BY c.id, c.nombre, c.ciudad ORDER BY total_comprado DESC;

Lo que aprendiste

  • JOIN combina filas de dos o más tablas usando una columna en común (ON)
  • INNER JOIN devuelve solo filas con coincidencia en ambas tablas
  • LEFT JOIN devuelve todas las filas de la tabla izquierda, con NULL donde no hay coincidencia
  • RIGHT JOIN devuelve todas las filas de la derecha — casi siempre reemplazable por LEFT JOIN
  • Los alias (c, p) hacen el código más limpio y son necesarios cuando hay columnas con el mismo nombre
  • Siempre usá prefijo de tabla (c.nombre, p.monto) para evitar columnas ambiguas
  • LEFT JOIN + WHERE IS NULL encuentra filas sin coincidencia en la otra tabla
  • JOIN + GROUP BY + funciones de agregado es la combinación más potente para reportes
  • COALESCE reemplaza NULL por un valor por defecto en resultados de LEFT JOIN
