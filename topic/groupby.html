Dominando el GROUP BY
en MySQL

Las funciones de agregado calculan sobre todas las filas. GROUP BY las divide en grupos para calcular por separado: total por ciudad, promedio por vendedor, conteo por categoría.

¿Qué es GROUP BY?

GROUP BY agrupa las filas que tienen el mismo valor en una o varias columnas, y luego aplica funciones de agregado (COUNT, SUM, AVG, MAX, MIN) a cada grupo por separado.

📦
Analogía de las cajas de correo Imagina que tenés 100 cartas mezcladas. GROUP BY es como el cartero que las separa por barrio antes de repartirlas: primero las agrupa (Quito, Guayaquil, Cuenca), y luego conta cuántas hay en cada grupo. Sin agrupar, solo podrías contar el total de todas.

Sin GROUP BY, las funciones de agregado devuelven un único resultado para toda la tabla. Con GROUP BY, devuelven un resultado por cada grupo.

Sintaxis básica

SELECT
Columna(s) agrupadas + funciones de agregado. Solo podés seleccionar columnas que estén en GROUP BY o dentro de una función.
FROM
La tabla de donde vienen los datos.
WHERE
Filtro de filas (opcional) — se aplica ANTES de agrupar.
GROUP BY
Columna por la que agrupar. Cada valor único genera una fila de resultado.
HAVING
Filtro de grupos (opcional) — se aplica DESPUÉS de agrupar.
ORDER BY
Ordenar el resultado (opcional).
Contar ventas por vendedor
SELECT vendedor, COUNT(*) AS total_ventas FROM ventas GROUP BY vendedor;
Resultado →
vendedortotal_ventas
Ana Torres3
Luis Mora3
Carmen Ríos2

8 filas → 3 filas GROUP BY condensó las 8 filas originales en 3 — una por cada vendedor único. La función COUNT(*) contó cuántas filas había en cada grupo.

La regla más importante de GROUP BY En el SELECT solo podés poner columnas que estén en el GROUP BY, o columnas dentro de una función de agregado. Todo lo demás genera un error o resultados impredecibles.
✗ Error — columna no agrupada
SELECT vendedor, producto, COUNT(*) FROM ventas GROUP BY vendedor; -- "producto" no está en -- GROUP BY ni en una función
✓ Correcto
SELECT vendedor, COUNT(*) AS ventas FROM ventas GROUP BY vendedor; -- Solo "vendedor" (en GROUP BY) -- y COUNT (función de agregado)

GROUP BY con funciones de agregado

La potencia de GROUP BY aparece al combinarlo con SUM, AVG, MAX, MIN y COUNT para obtener estadísticas por grupo.

Total vendido por ciudad
SELECT ciudad, COUNT(*) AS num_ventas, SUM(monto) AS total, ROUND(AVG(monto), 2) AS promedio, MAX(monto) AS mayor_venta FROM ventas GROUP BY ciudad;

Agrupar por múltiples columnas

Podés agrupar por más de una columna. En ese caso, cada combinación única de valores forma un grupo separado.

📊
Analogía de la tabla dinámica Es como una tabla dinámica de Excel con filas y columnas. Si agrupás por vendedor y ciudad a la vez, obtenés una fila para "Ana Torres en Quito", otra para "Luis Mora en Guayaquil", etc. Cada combinación única es un grupo.
Ventas por vendedor y ciudad
SELECT vendedor, ciudad, COUNT(*) AS ventas, SUM(monto) AS total FROM ventas GROUP BY vendedor, ciudad;
Resultado →
vendedorciudadventastotal
Ana TorresQuito31610.00
Luis MoraGuayaquil31495.00
Carmen RíosCuenca2175.00

Filtrar grupos con HAVING

HAVING es el WHERE de los grupos. Permite filtrar el resultado después de que GROUP BY ya agrupó las filas. Solo podés usar HAVING con funciones de agregado o columnas que estén en GROUP BY.

🎯
Analogía del filtro de calidad Imagina que fabricás cajas de manzanas. GROUP BY separa las manzanas por tamaño (pequeño, mediano, grande). HAVING es el control de calidad al final: "solo me quedan las cajas que tienen más de 10 manzanas". El filtro se aplica sobre los grupos ya formados, no sobre las manzanas individuales.
Solo ciudades con más de 2 ventas
SELECT ciudad, COUNT(*) AS total_ventas FROM ventas GROUP BY ciudad HAVING COUNT(*) > 2;
Resultado → (Cuenca queda excluida porque solo tiene 2 ventas)
ciudadtotal_ventas
Quito3
Guayaquil3
Vendedores con total vendido mayor a $1000
SELECT vendedor, SUM(monto) AS total FROM ventas GROUP BY vendedor HAVING SUM(monto) > 1000;
Resultado →
vendedortotal
Ana Torres1610.00
Luis Mora1495.00

WHERE vs HAVING — la diferencia clave

Es la duda más común con GROUP BY. La regla es simple: WHERE filtra filas antes de agrupar, HAVING filtra grupos después de agrupar.

WHERE — filtra filas (antes)
SELECT ciudad, COUNT(*) FROM ventas WHERE activo = 1 GROUP BY ciudad; -- Primero descarta filas -- donde activo = 0, -- luego agrupa el resto
HAVING — filtra grupos (después)
SELECT ciudad, COUNT(*) FROM ventas GROUP BY ciudad HAVING COUNT(*) > 2; -- Primero agrupa todas -- las filas, luego descarta -- grupos con 2 o menos
WHEREHAVING
¿Qué filtra?Filas individualesGrupos ya formados
¿Cuándo se aplica?Antes de GROUP BYDespués de GROUP BY
¿Con funciones?No puedeSí puede
EjemploWHERE activo = 1HAVING COUNT(*) > 2
Usando WHERE y HAVING juntos
-- Solo ventas activas (WHERE filtra filas) -- Solo ciudades con total > 1000 (HAVING filtra grupos) SELECT ciudad, COUNT(*) AS ventas, SUM(monto) AS total FROM ventas WHERE activo = 1 GROUP BY ciudad HAVING SUM(monto) > 1000;

GROUP BY + ORDER BY

Podés ordenar los grupos resultantes con ORDER BY. Se puede ordenar por cualquier columna del SELECT, incluyendo las funciones de agregado.

Ranking de vendedores por total vendido
SELECT vendedor, COUNT(*) AS num_ventas, SUM(monto) AS total FROM ventas GROUP BY vendedor ORDER BY total DESC;
Resultado → ordenado de mayor a menor total

🥈Luis Mora31495.00

Errores comunes

1. Seleccionar columna que no está en GROUP BY

✗ Error
SELECT vendedor, ciudad, COUNT(*) FROM ventas GROUP BY vendedor; -- "ciudad" no está en -- GROUP BY ni en función
✓ Correcto
SELECT vendedor, ciudad, COUNT(*) FROM ventas GROUP BY vendedor, ciudad; -- Ambas columnas están -- en GROUP BY ✓

2. Usar función de agregado en WHERE

✗ Error
SELECT ciudad, COUNT(*) FROM ventas WHERE COUNT(*) > 2 GROUP BY ciudad; -- WHERE no puede usar -- funciones de agregado
✓ Correcto
SELECT ciudad, COUNT(*) FROM ventas GROUP BY ciudad HAVING COUNT(*) > 2; -- HAVING es para filtrar -- con funciones ✓

3. Poner HAVING antes de GROUP BY

✗ Error de orden
SELECT ciudad, COUNT(*) FROM ventas HAVING COUNT(*) > 2 GROUP BY ciudad; -- HAVING siempre va -- después de GROUP BY
✓ Orden correcto
SELECT ciudad, COUNT(*) FROM ventas GROUP BY ciudad HAVING COUNT(*) > 2; -- GROUP BY → HAVING ✓

Resumen: GROUP BY completo

Todo junto — reporte completo de ventas activas
-- Reporte: vendedores activos con más de 1 venta, -- ordenados por total de mayor a menor SELECT vendedor, ciudad, COUNT(*) AS num_ventas, ROUND(SUM(monto), 2) AS total_vendido, ROUND(AVG(monto), 2) AS promedio, MAX(monto) AS mayor_venta FROM ventas WHERE activo = 1 GROUP BY vendedor, ciudad HAVING COUNT(*) > 1 ORDER BY total_vendido DESC;

Lo que aprendiste

  • GROUP BY agrupa filas con el mismo valor y aplica funciones de agregado por grupo
  • Solo podés poner en SELECT columnas que estén en GROUP BY o dentro de una función
  • Podés agrupar por varias columnas: cada combinación única forma un grupo
  • WHERE filtra filas antes de agrupar — no puede usar funciones de agregado
  • HAVING filtra grupos después de agrupar — sí puede usar funciones de agregado
  • ORDER BY ordena el resultado final y puede usar columnas con alias
  • El orden correcto es: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
  • GROUP BY + SUM/COUNT/AVG/MAX/MIN es la combinación más potente para reportes
