Dominando el DELETE
en MySQL

Borrar datos parece simple, pero un DELETE mal escrito puede vaciar una tabla entera en un instante. Aprendé a usarlo con precisión y seguridad.

¿Qué es DELETE?

DELETE FROM elimina filas de una tabla de forma permanente. A diferencia de UPDATE (que modifica datos) o SELECT (que solo los lee), DELETE los borra definitivamente sin posibilidad de recuperación, a menos que tengas una copia de seguridad.

🗑️
Analogía de la papelera En tu computadora, cuando eliminás un archivo va a la papelera — podés recuperarlo. DELETE en MySQL es diferente: es como borrar el archivo y vaciar la papelera al mismo tiempo. No hay Ctrl+Z. Por eso hay que usarlo siempre con cuidado y verificación previa.

Se usa cuando necesitás eliminar registros obsoletos, usuarios que cancelaron su cuenta, productos descontinuados, o limpiar datos de prueba en desarrollo.

Sintaxis básica

DELETE tiene una estructura muy simple. Solo necesitás decirle de qué tabla y con qué condición.

DELETE FROM
¿De qué tabla? El nombre de la tabla de donde se eliminarán las filas.
WHERE
¿Qué filas eliminar? La condición que deben cumplir. Sin WHERE se borran TODAS las filas.
Eliminar a Pedro Vega (id = 4)
DELETE FROM empleados WHERE id = 4;

💡
Usá siempre el id en WHERE El id identifica una sola fila de forma única. Es la forma más segura de asegurarte de que solo se borre exactamente el registro que querés.

El peligro de olvidar WHERE

Este es el error más grave que podés cometer con DELETE. Sin WHERE, MySQL borra absolutamente todas las filas de la tabla sin preguntar ni avisar.

✗ Sin WHERE — catastrófico
DELETE FROM empleados; -- Resultado: la tabla -- queda completamente -- vacía. 0 filas.
✓ Con WHERE — seguro
DELETE FROM empleados WHERE id = 4; -- Solo se borra -- la fila con id = 4. -- El resto queda intacto.
⚠️
Regla absoluta: nunca DELETE sin WHERE en producción Un DELETE sin WHERE en una tabla con millones de registros los borra todos en segundos. No hay deshacer. Verificá siempre con un SELECT antes de ejecutar el DELETE.
🏎️
Analogía del freno de fórmula 1 Un DELETE sin WHERE es como un auto de Fórmula 1 sin frenos: increíblemente rápido y totalmente fuera de control. El WHERE es el sistema de frenos. Sin él, el resultado es el choque inevitable.

Condiciones en WHERE

WHERE en DELETE funciona exactamente igual que en SELECT. Podés usar cualquier operador de comparación para apuntar a exactamente las filas que querés borrar.

Eliminar empleados inactivos (activo = 0)
DELETE FROM empleados WHERE activo = 0;

Eliminar múltiples filas

DELETE puede eliminar muchas filas a la vez si la condición WHERE coincide con varias. Esto es útil para limpiezas masivas de datos.

🧹
Analogía de la limpieza Es como contratar a un servicio de limpieza con instrucciones específicas: "eliminar todo lo que esté en mal estado en el almacén". No vas objeto por objeto — dás la instrucción una vez y el servicio aplica el criterio a todos.
Limpiar datos de prueba
DELETE FROM usuarios WHERE email LIKE '%@test.com'; -- Elimina todos los usuarios -- registrados con emails de prueba
Eliminar registros antiguos
DELETE FROM logs WHERE fecha < '2023-01-01'; -- Borra todos los logs -- anteriores al año 2023
LIMIT: borrar máximo N filas por seguridad
DELETE FROM empleados WHERE activo = 0 LIMIT 10; -- Aunque haya 50 inactivos, -- solo borra los primeros 10
🛡️
LIMIT como red de seguridad Agregar LIMIT N al DELETE pone un techo máximo de filas a eliminar. Ideal cuando no estás 100% seguro de cuántas filas cumplirán la condición.

DELETE vs TRUNCATE

Cuando querés vaciar una tabla entera, hay dos opciones. Entender la diferencia es importante.

DELETE FROM tablaTRUNCATE TABLE tabla
VelocidadMás lento (fila por fila)Mucho más rápido
WHERESí admite WHERENo admite WHERE
AUTO_INCREMENTEl contador continúaEl contador se reinicia a 1
TransaccionesSe puede deshacer con ROLLBACKNo se puede deshacer
TriggersLos activaNo los activa
Uso típicoBorrar filas específicasVaciar tabla completamente
DELETE sin WHERE
DELETE FROM empleados; -- Lento, pero el id -- AUTO_INCREMENT -- continúa desde -- donde estaba
TRUNCATE (más eficiente)
TRUNCATE TABLE empleados; -- Rápido, vacía todo -- y reinicia el id -- AUTO_INCREMENT a 1
📌
¿Cuándo usar cada uno? Usá DELETE + WHERE para borrar filas específicas. Usá TRUNCATE solo cuando querés vaciar la tabla entera de forma rápida, como al limpiar datos de prueba en desarrollo.

El proceso seguro para hacer un DELETE

Nunca ejecutés un DELETE sin seguir estos pasos, especialmente en bases de datos de producción.

1

Verificá con SELECT primero

Ejecutá el mismo WHERE pero con SELECT para ver exactamente qué filas vas a borrar.

2

Contá las filas afectadas

Usá SELECT COUNT(*) para saber cuántas filas se van a eliminar. ¿Es el número que esperabas?

3

Usá una transacción

Envolvé el DELETE en START TRANSACTION para poder hacer ROLLBACK si algo sale mal.

4

Confirmá con COMMIT

Solo después de verificar que el resultado es correcto, ejecutá COMMIT para hacer permanente el borrado.

Flujo completo y seguro
-- Paso 1: ver qué se va a borrar SELECT * FROM empleados WHERE activo = 0; -- Paso 2: contar las filas SELECT COUNT(*) FROM empleados WHERE activo = 0; -- Paso 3: iniciar transacción START TRANSACTION; DELETE FROM empleados WHERE activo = 0; -- Paso 4: verificar resultado SELECT * FROM empleados; -- Si está bien → confirmar COMMIT; -- Si algo salió mal → deshacer -- ROLLBACK;

Errores comunes

1. WHERE con texto ambiguo en vez de id

✗ Ambiguo
DELETE FROM empleados WHERE nombre = 'Ana'; -- ¿Cuántas "Ana" -- hay en la tabla?
✓ Preciso
DELETE FROM empleados WHERE id = 1; -- Solo una fila -- con ese id ✓

2. Olvidar que DELETE no tiene UNDO

✗ Sin respaldo
DELETE FROM pedidos WHERE año = 2020; -- Sin backup ni -- transacción. Si -- te equivocaste, -- no hay vuelta atrás.
✓ Con transacción
START TRANSACTION; DELETE FROM pedidos WHERE año = 2020; -- Revisás y decidís: -- COMMIT o ROLLBACK

3. Intentar borrar con una clave foránea activa

🔗
Error: Cannot delete or update a parent row Si la fila que intentás borrar está referenciada por otra tabla (clave foránea), MySQL no te dejará. Primero debés borrar o actualizar las filas hijas, o desactivar temporalmente la restricción con SET FOREIGN_KEY_CHECKS = 0.

Resumen: DELETE completo

Todo junto en un ejemplo real
-- 1. Ver qué se va a borrar SELECT nombre, activo FROM empleados WHERE activo = 0 AND departamento = 'RRHH'; -- 2. Contar filas afectadas SELECT COUNT(*) FROM empleados WHERE activo = 0 AND departamento = 'RRHH'; -- 3. Ejecutar con transacción START TRANSACTION; DELETE FROM empleados WHERE activo = 0 AND departamento = 'RRHH' LIMIT 5; COMMIT;

Lo que aprendiste

  • DELETE FROM elimina filas permanentemente — no hay deshacer sin transacción
  • WHERE determina qué filas se eliminan — sin WHERE se borran todas
  • Usá siempre el id en WHERE para borrar exactamente una fila
  • IN permite eliminar varias filas específicas en una sola sentencia
  • LIMIT pone un tope máximo de filas eliminadas por seguridad
  • Verificá siempre con SELECT y COUNT(*) antes de ejecutar el DELETE
  • Usá START TRANSACTION para poder hacer ROLLBACK si algo sale mal
  • TRUNCATE vacía la tabla completa más rápido y reinicia el AUTO_INCREMENT
