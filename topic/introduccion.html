Introducción a SQL
y bases de datos

Antes de consultar o modificar datos, necesitas saber dónde viven y cómo están organizados. Esta guía te explica qué es una base de datos, cómo se estructuran las tablas y cómo crearlas y modificarlas desde cero.

¿Qué es SQL?

SQL (Structured Query Language) es el lenguaje estándar para comunicarse con bases de datos relacionales. Con SQL puedes crear estructuras, insertar datos, consultarlos, actualizarlos y eliminarlos.

💬
Analogía del idioma SQL es como aprender un idioma para hablar con una base de datos. Tú escribes instrucciones en SQL y la base de datos las entiende y responde. MySQL, PostgreSQL y SQLite son distintos "dialectos" del mismo idioma base.

SQL se divide en tres grupos de comandos principales:

🏗️
DDL
Data Definition Language. Crea y modifica estructuras: CREATE, ALTER, DROP.
✏️
DML
Data Manipulation Language. Manipula datos: INSERT, UPDATE, DELETE, SELECT.
🔐
DCL
Data Control Language. Controla permisos: GRANT, REVOKE.
💾
TCL
Transaction Control Language. Maneja transacciones: COMMIT, ROLLBACK.

¿Qué es una base de datos?

Una base de datos es un sistema organizado para almacenar, gestionar y recuperar información de forma eficiente. En bases de datos relacionales, la información se guarda en tablas con filas y columnas.

📁
Analogía del archivador Una base de datos es como un archivador de oficina. Dentro hay carpetas (tablas), cada carpeta tiene hojas (filas), y cada hoja tiene campos llenados (columnas). Todo está organizado para encontrar cualquier documento rápidamente.
Crear una base de datos y seleccionarla
-- Crear la base de datos CREATE DATABASE tienda; -- Seleccionarla para trabajar en ella USE tienda; -- Ver todas las bases de datos SHOW DATABASES;

¿Qué es una tabla?

Una tabla es la unidad básica de almacenamiento en SQL. Se organiza en columnas (campos) y filas (registros). Cada columna tiene un nombre y un tipo de dato específico.

📊
Analogía de la hoja de Excel Una tabla es como una hoja de cálculo: la primera fila define los encabezados (columnas) y cada fila siguiente es un registro. La diferencia es que en SQL cada columna tiene un tipo de dato estricto — no podés mezclar texto con números en la misma columna.

Tipos de datos en MySQL

Cada columna debe declarar qué tipo de dato acepta. Esto garantiza consistencia y eficiencia en el almacenamiento.

Tipos numéricos

DECIMAL(p,d)Número con decimales exactos19.99, 3410.50
FLOATNúmero con decimales aproximados3.14, 2.718

BIGINTEntero muy grande9999999999

Tipos de texto

Tipos de fecha y hora

TipoFormatoEjemplo

💡
Regla general para elegir tipos Usa INT para identificadores, VARCHAR para textos cortos, DECIMAL para precios y valores monetarios, DATE para fechas, y TINYINT(1) como booleano (0 = falso, 1 = verdadero).

Crear una tabla — CREATE TABLE

CREATE TABLE define la estructura de una nueva tabla: su nombre, columnas, tipos de datos y restricciones.

CREATE TABLE
Comando principal. Le dice a MySQL que vas a crear una nueva tabla.
nombre_tabla
El nombre de la tabla. Usa nombres descriptivos en minúsculas y sin espacios.
( ... )
Definición de columnas. Cada columna se define con: nombre tipo_de_dato restricciones.
tipo_dato
El tipo de la columna. INT, VARCHAR, DECIMAL, DATE, etc.
Crear tabla de empleados
CREATE TABLE empleados ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(100) NOT NULL, ciudad VARCHAR(80), departamento VARCHAR(60), salario DECIMAL(10,2) NOT NULL, activo TINYINT DEFAULT 1, fecha_ingreso DATE );
Crear tabla de productos
CREATE TABLE productos ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(100) NOT NULL, descripcion TEXT, precio DECIMAL(8,2) NOT NULL, stock INT DEFAULT 0, categoria VARCHAR(50), creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
Ver las tablas existentes y la estructura
-- Ver todas las tablas de la base de datos SHOW TABLES; -- Ver la estructura de una tabla DESCRIBE empleados; -- O también: SHOW COLUMNS FROM empleados;

Restricciones — CONSTRAINTS

Las restricciones son reglas que se aplican a las columnas para garantizar la integridad de los datos. Se definen al crear la tabla o se agregan después.

RestricciónSignificadoEjemplo
PRIMARY KEYIdentifica cada fila de forma únicaid INT PRIMARY KEY
AUTO_INCREMENTGenera el valor automáticamenteid INT AUTO_INCREMENT
NOT NULLEl campo no puede estar vacíonombre VARCHAR(100) NOT NULL
DEFAULTValor por defecto si no se especificaactivo TINYINT DEFAULT 1
UNIQUEEl valor debe ser único en la tablaemail VARCHAR(100) UNIQUE
FOREIGN KEYReferencia a otra tablaVer más abajo
🔑
PRIMARY KEY + AUTO_INCREMENT Siempre declara una columna id con INT AUTO_INCREMENT PRIMARY KEY en tus tablas. Esto le da a cada fila un identificador único que se asigna automáticamente.
Tabla con FOREIGN KEY — clave foránea
CREATE TABLE pedidos ( id INT AUTO_INCREMENT PRIMARY KEY, empleado_id INT NOT NULL, producto VARCHAR(100), monto DECIMAL(8,2), fecha DATE, FOREIGN KEY (empleado_id) REFERENCES empleados(id) );

Agregar columnas — ALTER TABLE ADD

Después de crear una tabla, podés agregar nuevas columnas sin perder los datos existentes usando ALTER TABLE ... ADD.

➕
Analogía del formulario ampliado Es como agregar un campo nuevo a un formulario ya impreso. Los formularios anteriores quedarán con ese campo en blanco (NULL o con el DEFAULT), pero los nuevos ya lo tendrán disponible.
Agregar columnas a tabla existente
-- Agregar una columna al final ALTER TABLE empleados ADD email VARCHAR(150); -- Agregar con restricción NOT NULL y DEFAULT ALTER TABLE empleados ADD telefono VARCHAR(20) DEFAULT 'Sin teléfono'; -- Agregar al inicio de la tabla ALTER TABLE empleados ADD codigo VARCHAR(10) FIRST; -- Agregar después de una columna específica ALTER TABLE empleados ADD apellido VARCHAR(100) AFTER nombre; -- Agregar varias columnas a la vez ALTER TABLE productos ADD peso DECIMAL(6,2), ADD color VARCHAR(30), ADD activo TINYINT DEFAULT 1;

Modificar columnas — ALTER TABLE MODIFY

Para cambiar el tipo de dato, el tamaño o las restricciones de una columna existente, usás ALTER TABLE ... MODIFY.

Modificar columnas existentes
-- Cambiar el tipo o tamaño ALTER TABLE empleados MODIFY nombre VARCHAR(200) NOT NULL; -- Cambiar a NOT NULL y agregar DEFAULT ALTER TABLE empleados MODIFY ciudad VARCHAR(80) NOT NULL DEFAULT 'Sin ciudad'; -- Cambiar tipo de columna ALTER TABLE productos MODIFY precio DECIMAL(12,2) NOT NULL; -- Renombrar columna y cambiar tipo (CHANGE) ALTER TABLE empleados CHANGE telefono celular VARCHAR(15);
MODIFY — cambia tipo/restricción
ALTER TABLE empleados MODIFY nombre VARCHAR(200) NOT NULL; -- Cambia tamaño -- Mantiene el nombre
CHANGE — renombra y cambia tipo
ALTER TABLE empleados CHANGE telefono celular VARCHAR(15); -- Renombra la columna -- Y cambia el tipo
⚠️
Cuidado al cambiar tipos de datos Si cambias una columna de VARCHAR a INT y hay datos de texto, MySQL puede perder información. Siempre verificá los datos existentes antes de modificar el tipo de una columna en producción.

Eliminar columnas — ALTER TABLE DROP

Para eliminar una columna de una tabla existente usás ALTER TABLE ... DROP COLUMN. Esta acción borra la columna y todos sus datos de forma permanente.

Eliminar columnas
-- Eliminar una columna ALTER TABLE empleados DROP COLUMN codigo; -- Eliminar varias columnas a la vez ALTER TABLE empleados DROP COLUMN peso, DROP COLUMN color;
🗑️
DROP COLUMN es permanente Al eliminar una columna, todos sus datos se pierden inmediatamente. No hay deshacer. Hacé siempre un backup antes de ejecutar DROP COLUMN en producción.

Renombrar tabla — RENAME TABLE

Podés cambiar el nombre de una tabla existente sin perder ningún dato usando RENAME TABLE o ALTER TABLE ... RENAME TO.

Renombrar tablas
-- Forma 1: RENAME TABLE RENAME TABLE empleados TO staff; -- Forma 2: ALTER TABLE ALTER TABLE productos RENAME TO catalogo; -- Renombrar varias tablas a la vez RENAME TABLE pedidos TO ordenes, clientes TO customers;

Eliminar tabla — DROP TABLE

DROP TABLE elimina completamente una tabla con todos sus datos y estructura. Es irreversible.

Eliminar tablas
-- Eliminar una tabla DROP TABLE empleados; -- Eliminar solo si existe (sin error) DROP TABLE IF EXISTS empleados; -- Eliminar varias tablas DROP TABLE tabla1, tabla2, tabla3; -- Vaciar la tabla sin borrar la estructura TRUNCATE TABLE empleados;

Lo que aprendiste

  • SQL es el lenguaje estándar para bases de datos relacionales
  • Una base de datos contiene tablas; una tabla contiene filas y columnas
  • Cada columna tiene un tipo de dato: INT, VARCHAR, DECIMAL, DATE, etc.
  • CREATE TABLE crea una tabla nueva con su estructura y restricciones
  • PRIMARY KEY + AUTO_INCREMENT identifica cada fila de forma única
  • NOT NULL impide que un campo quede vacío; DEFAULT asigna un valor automático
  • ALTER TABLE ADD agrega nuevas columnas sin perder datos existentes
  • ALTER TABLE MODIFY cambia el tipo o restricciones de una columna
  • ALTER TABLE CHANGE renombra una columna y cambia su tipo al mismo tiempo
  • ALTER TABLE DROP COLUMN elimina una columna de forma permanente
  • RENAME TABLE cambia el nombre de una tabla sin afectar sus datos
  • DROP TABLE elimina la tabla completa; TRUNCATE solo vacía sus datos
