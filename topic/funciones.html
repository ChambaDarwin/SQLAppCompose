Funciones de agregado
en MySQL

Con SELECT puedes leer filas, pero ¿cómo calculas el total, el promedio o el máximo de un conjunto de datos? Para eso existen las funciones de agregado: COUNT, SUM, AVG, MAX y MIN.

¿Qué son las funciones de agregado?

Las funciones de agregado toman múltiples filas y las condensan en un solo valor. En lugar de devolver fila por fila, calculan un resultado sobre todo el conjunto.

🧮
Analogía de la hoja de cálculo Imagina una hoja de Excel con 1000 filas de ventas. En vez de leerlas una a una, usás una fórmula: =SUMA(ventas), =PROMEDIO(ventas), =MAX(ventas). Las funciones de agregado en MySQL son exactamente esas fórmulas, pero dentro de una consulta SQL.

Las cinco funciones principales son:

COUNT()
Cuenta cuántas filas existen
SUM()
Suma todos los valores
AVG()
Calcula el promedio
MAX()
Devuelve el valor mayor
MIN()
Devuelve el valor menor

COUNT() — Contar filas

COUNT cuenta cuántas filas hay en el resultado. Es una de las funciones más usadas en SQL.

🔢
Analogía del inventario COUNT es como contar cuántos productos hay en una bodega. No te dice el precio ni el peso, solo cuántos hay. Una respuesta: un número.
COUNT(*)
Cuenta todas las filas, incluyendo las que tienen NULL en alguna columna.
COUNT(col)
Cuenta filas donde esa columna NO es NULL. Ignora los valores nulos.
COUNT(DISTINCT)
Cuenta valores únicos — sin repetidos.
¿Cuántas ventas hay en total?
SELECT COUNT(*) AS total_ventas FROM ventas;
Resultado →
total_ventas
8
¿Cuántos vendedores distintos hay?
SELECT COUNT(DISTINCT vendedor) AS vendedores_unicos FROM ventas;
Resultado →
vendedores_unicos
3

SUM() — Sumar valores

SUM suma todos los valores de una columna numérica. Ignora los NULL automáticamente.

💰
Analogía de la caja registradora SUM es como sumar todos los tickets de una caja al final del día. No importa cuántos sean, te dice el total acumulado en un solo número.
¿Cuánto se vendió en total?
SELECT SUM(monto) AS total_vendido FROM ventas;
Resultado →
total_vendido
3280.00
Total vendido solo en Quito
SELECT SUM(monto) AS ventas_quito FROM ventas WHERE ciudad = 'Quito';
Resultado → (1200 + 350 + 60)
ventas_quito
1610.00

AVG() — Calcular el promedio

AVG divide la suma de todos los valores entre la cantidad de filas no nulas. Siempre ignora los NULL al calcular.

📊
Analogía de las notas escolares AVG es como calcular tu nota promedio: sumás todas tus calificaciones y las dividís entre la cantidad de materias. MySQL hace ese cálculo automáticamente sobre toda la columna.
Monto promedio por venta
SELECT AVG(monto) AS promedio_venta FROM ventas;
Resultado → (3280 ÷ 8)
promedio_venta
410.00
Redondear el promedio a 2 decimales
SELECT ROUND(AVG(monto), 2) AS promedio FROM ventas;
💡
AVG ignora los NULL Si una fila tiene NULL en la columna calculada, AVG no la cuenta ni en la suma ni en el divisor. Esto puede hacer que el promedio sea más alto de lo esperado si hay muchos nulls.

MAX() y MIN() — Mayor y menor

MAX devuelve el valor más alto y MIN el más bajo de una columna. Funcionan con números, textos y fechas.

🏔️
Analogía del termómetro MAX es la temperatura máxima registrada en el día, MIN es la mínima. No te dan todas las mediciones, solo los extremos. Perfecto para saber el rango de tus datos.
Venta más alta y más baja
SELECT MAX(monto) AS venta_mayor, MIN(monto) AS venta_menor FROM ventas;
Resultado →
venta_mayorventa_menor
1200.0025.00
MAX y MIN también funcionan con texto
-- MAX con texto devuelve el último en orden alfabético SELECT MAX(vendedor) AS ultimo_alfabetico, MIN(vendedor) AS primero_alfabetico FROM ventas; -- Resultado: 'Sofía León' y 'Ana Torres'

Usar varias funciones juntas

Podés usar todas las funciones que necesites en un mismo SELECT para obtener un resumen completo en una sola consulta.

Resumen completo de ventas
SELECT COUNT(*) AS total_registros, COUNT(DISTINCT vendedor) AS vendedores, SUM(monto) AS total_vendido, AVG(monto) AS promedio, MAX(monto) AS mayor_venta, MIN(monto) AS menor_venta FROM ventas;
Resultado →
total_registrosvendedorestotal_vendidopromediomayor_ventamenor_venta
8 3 3280.00 410.00 1200.00 25.00

Funciones con WHERE

WHERE filtra las filas antes de que la función las procese. Así podés calcular estadísticas para subconjuntos específicos de datos.

🔬
WHERE filtra primero, la función calcula después El orden de ejecución interno es: FROM → WHERE (filtrar filas) → SELECT (aplicar funciones). La función nunca "ve" las filas que WHERE descartó.
Estadísticas solo de ventas activas en Quito
SELECT COUNT(*) AS ventas, SUM(monto) AS total, AVG(monto) AS promedio FROM ventas WHERE ciudad = 'Quito' AND activo = 1;
Resultado →
ventastotalpromedio
3 1610.00 536.67
Comparar ventas por ciudad
-- Quito SELECT 'Quito' AS ciudad, SUM(monto) AS total FROM ventas WHERE ciudad = 'Quito' UNION ALL -- Guayaquil SELECT 'Guayaquil', SUM(monto) FROM ventas WHERE ciudad = 'Guayaquil';

Alias con AS

Por defecto, MySQL nombra las columnas resultado con el nombre de la función (COUNT(*), SUM(monto)). Con AS les das nombres legibles.

Sin AS (feo)Con AS (legible)
COUNT(*)total_ventas
SUM(monto)ingresos_totales
AVG(monto)ticket_promedio
MAX(monto)venta_maxima
Siempre usá AS con funciones de agregado
SELECT COUNT(*) AS 'Total de ventas', SUM(monto) AS 'Ingresos totales', AVG(monto) AS 'Ticket promedio' FROM ventas;

Funciones y valores NULL

Las funciones de agregado tienen un comportamiento especial con NULL. Es importante entenderlo para evitar resultados inesperados.

Función¿Qué hace con NULL?
COUNT(*)Los cuenta — incluye filas con NULL
COUNT(col)Los ignora — no cuenta filas donde esa col es NULL
SUM(col)Los ignora — no los suma
AVG(col)Los ignora en suma Y en el conteo del divisor
MAX(col)Los ignora — no los considera
MIN(col)Los ignora — no los considera
⚠️
COUNT(*) vs COUNT(columna): la diferencia clave Si tenés 100 filas pero 10 tienen NULL en la columna monto: COUNT(*) devuelve 100, pero COUNT(monto) devuelve 90. Usá COUNT(*) para contar filas, COUNT(col) para contar valores existentes.

Ejemplo combinando funciones de texto y número
SELECT UPPER(vendedor) AS nombre, COUNT(*) AS num_ventas, ROUND(SUM(monto), 2) AS total, ROUND(AVG(monto), 2) AS promedio FROM ventas WHERE activo = 1 GROUP BY vendedor ORDER BY total DESC;
📌
GROUP BY aparece aquí — ¿qué es? El ejemplo de arriba usa GROUP BY para calcular estadísticas por cada vendedor. GROUP BY es el siguiente tema de la guía y es el complemento natural de las funciones de agregado.

Resumen: funciones completo

Todo junto en un ejemplo real
-- Reporte completo de ventas activas SELECT COUNT(*) AS 'Total ventas', COUNT(DISTINCT vendedor) AS 'Vendedores', ROUND(SUM(monto), 2) AS 'Ingresos totales', ROUND(AVG(monto), 2) AS 'Ticket promedio', MAX(monto) AS 'Venta más alta', MIN(monto) AS 'Venta más baja' FROM ventas WHERE activo = 1;

Lo que aprendiste

  • Las funciones de agregado condensan múltiples filas en un solo resultado
  • COUNT(*) cuenta todas las filas; COUNT(col) ignora los NULL de esa columna
  • SUM() suma valores numéricos ignorando NULL
  • AVG() calcula el promedio ignorando NULL en suma Y en el divisor
  • MAX() y MIN() devuelven el valor más alto y más bajo; funcionan con texto y fechas
  • Podés usar varias funciones en el mismo SELECT para un resumen completo
  • WHERE filtra las filas antes de que la función las procese
  • Usá siempre AS para darle nombres legibles a los resultados
  • ROUND(), UPPER(), NOW() y otras funciones complementan las de agregado
