Dominando el ORDER BY
en MySQL

Los datos en una base de datos no tienen un orden natural. ORDER BY es el comando que te permite presentarlos exactamente como los necesitas: de mayor a menor, alfabéticamente, por fecha, o como tú decidas.

¿Qué es ORDER BY?

ORDER BY es una cláusula del SELECT que ordena las filas del resultado de una consulta. Por sí solo, MySQL no garantiza ningún orden específico al devolver datos — ORDER BY es la única forma confiable de obtener resultados ordenados.

📚
Analogía de la biblioteca Imagina una biblioteca con miles de libros apilados sin orden. Si le pides al bibliotecario "dame todos los libros de ciencia", te los traerá… pero en cualquier orden. Si además le dices "ordénalos por año de publicación, del más nuevo al más viejo", ahí interviene ORDER BY: mismos datos, presentación organizada.

ORDER BY solo afecta la presentación del resultado, nunca modifica los datos almacenados en la tabla.

ORDER BY siempre va al final de la consulta, después del WHERE si lo hay.

ASC y DESC

Son los dos modificadores de dirección. ASC (ascendente) es el valor por defecto, así que podés omitirlo. DESC (descendente) invierte el orden.

↑ ASC — de menor a mayor
SELECT nombre, salario FROM empleados ORDER BY salario ASC;
↓ DESC — de mayor a menor
SELECT nombre, salario FROM empleados ORDER BY salario DESC;
Resultado ASC →
Marco Díaz — 1750 Pedro Vega — 1850 Luis Mora — 1900 Sofía León — 2100 Ana Torres — 3410 Carmen Ríos — 3520
Resultado DESC →
Carmen Ríos — 3520 Ana Torres — 3410 Sofía León — 2100 Luis Mora — 1900 Pedro Vega — 1850 Marco Díaz — 1750
💡
ASC es el valor por defecto Si escribís ORDER BY salario sin especificar ASC o DESC, MySQL lo ordena de menor a mayor automáticamente.
🎵
Analogía de la lista de reproducción ASC es como ordenar tus canciones por duración de la más corta a la más larga. DESC es al revés: primero las más largas. La lista de canciones (los datos) no cambia, solo el orden en que aparecen.

Ordenar por múltiples columnas

Podés indicar varias columnas en ORDER BY separadas por comas. La segunda columna actúa como desempate cuando hay valores iguales en la primera.

🏆
Analogía del ranking de estudiantes En un concurso, primero ordenás a los estudiantes por nota. Si dos tienen la misma nota, los desempatás por apellido alfabéticamente. Eso es exactamente lo que hace ORDER BY con múltiples columnas.
Ordenar por departamento y luego por salario
SELECT nombre, departamento, salario FROM empleados ORDER BY departamento ASC, salario DESC;
Resultado → primero agrupa por departamento (A→Z), luego ordena salario de mayor a menor dentro de cada grupo

Cada columna puede tener su propia dirección
-- Ciudad A→Z, y dentro de cada ciudad -- salario de mayor a menor SELECT nombre, ciudad, salario FROM empleados ORDER BY ciudad ASC, salario DESC;

ORDER BY junto con WHERE

WHERE filtra primero qué filas incluir, y ORDER BY ordena después esas filas filtradas. El orden importa: WHERE siempre va antes de ORDER BY.

Empleados activos en Quito, ordenados por salario
SELECT nombre, ciudad, salario FROM empleados WHERE ciudad = 'Quito' AND activo = 1 ORDER BY salario DESC;
Resultado →
nombreciudadsalario
Carmen RíosQuito3520
Ana TorresQuito3410
Marco DíazQuito1750
📋
Orden correcto de las cláusulas SELECT → FROM → WHERE → ORDER BY → LIMIT. MySQL los ejecuta en un orden interno diferente, pero siempre hay que escribirlos en este orden o dará error de sintaxis.

ORDER BY junto con LIMIT

La combinación ORDER BY + LIMIT es muy poderosa: ordena los resultados y luego corta para obtener el "top N".

🥇
Analogía del podio En una maratón, todos los corredores terminan en algún orden. ORDER BY los ordena por tiempo. LIMIT 3 se queda solo con los tres primeros. Juntos forman el podio.
Top 3 salarios más altos
SELECT nombre, salario FROM empleados ORDER BY salario DESC LIMIT 3;
Resultado →
#nombresalario
🥇Carmen Ríos3520
🥈Ana Torres3410
🥉Sofía León2100
El empleado con menor salario
SELECT nombre, salario FROM empleados ORDER BY salario ASC LIMIT 1;
LIMIT + OFFSET para paginación
-- Página 1: filas 1 a 3 SELECT nombre, salario FROM empleados ORDER BY salario DESC LIMIT 3 OFFSET 0; -- Página 2: filas 4 a 6 SELECT nombre, salario FROM empleados ORDER BY salario DESC LIMIT 3 OFFSET 3;

Ordenar con valores NULL

Cuando una columna tiene valores NULL, MySQL los trata de forma especial al ordenar.

Dirección¿Dónde aparecen los NULL?
ASCLos NULL aparecen primero (antes de cualquier valor)
DESCLos NULL aparecen al final (después de cualquier valor)
Forzar NULLs al final en orden ASC
-- Truco: IS NULL devuelve 0 (no null) o 1 (null) -- Ordenar primero por ese valor (0 antes que 1) -- pone los NULL al final aunque sea ASC SELECT nombre, salario FROM empleados ORDER BY salario IS NULL ASC, salario ASC;
ℹ️
MySQL 8+ tiene NULLS FIRST / NULLS LAST En versiones modernas podés escribir directamente ORDER BY salario ASC NULLS LAST para controlar exactamente dónde aparecen los valores nulos.

ORDER BY según el tipo de dato

El comportamiento de ORDER BY varía según el tipo de dato de la columna:

TipoOrden ASCEjemplo
NuméricoDe menor a mayor número1, 5, 10, 100, 1000
Texto (VARCHAR)Alfabético (A → Z)Ana, Carmen, Luis, Marco
Fecha (DATE)De la más antigua a la más nueva2020-01-01, 2022-06-15, 2024-03-01
TINYINT / BIT0 antes que 10, 0, 1, 1, 1
⚠️
Cuidado con números guardados como texto Si una columna es VARCHAR pero contiene números, el orden será textual, no numérico: '10', '2', '20', '3' → porque '1' < '2' en texto. Para orden numérico correcto, la columna debe ser INT o DECIMAL.

Errores comunes

1. ORDER BY antes que WHERE

✗ Error de sintaxis
SELECT nombre, salario FROM empleados ORDER BY salario DESC WHERE activo = 1; -- WHERE debe ir antes -- de ORDER BY
✓ Correcto
SELECT nombre, salario FROM empleados WHERE activo = 1 ORDER BY salario DESC; -- WHERE → ORDER BY ✓

2. Ordenar por columna que no existe

✗ Error
SELECT nombre FROM empleados ORDER BY sueldo; -- "sueldo" no existe, -- la columna es "salario"
✓ Correcto
SELECT nombre FROM empleados ORDER BY salario; -- nombre exacto -- de la columna ✓

3. Esperar un orden sin ORDER BY

🎲
Sin ORDER BY el orden es impredecible MySQL puede devolver las filas en cualquier orden dependiendo del motor de almacenamiento, los índices y las operaciones recientes. Si necesitás un orden específico, siempre usá ORDER BY, nunca des el orden por garantizado.

Lo que aprendiste

  • ORDER BY ordena el resultado de una consulta, no los datos de la tabla
  • ASC ordena de menor a mayor (es el valor por defecto)
  • DESC ordena de mayor a menor
  • Podés ordenar por varias columnas: la segunda desempata a la primera
  • Cada columna en el ORDER BY puede tener su propia dirección (ASC o DESC)
  • ORDER BY siempre va después de WHERE y antes de LIMIT
  • ORDER BY + LIMIT es la forma estándar de obtener el "Top N"
  • Sin ORDER BY, el orden del resultado es impredecible
  • Los NULL aparecen primero con ASC y al final con DESC
