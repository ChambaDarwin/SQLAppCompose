Dominando el UPDATE
en MySQL

Los datos cambian constantemente: precios, direcciones, estados. UPDATE es el comando que mantiene tu base de datos al día sin borrar ni recrear registros.

¿Qué es UPDATE?

UPDATE modifica los valores de filas que ya existen en una tabla. A diferencia de INSERT (que crea filas nuevas) y DELETE (que las elimina), UPDATE simplemente cambia el contenido de columnas específicas.

✏️
Analogía del lápiz corrector Imagina una libreta con los datos de tus clientes escritos a lápiz. UPDATE es el borrador y el lápiz: tachás lo que estaba mal y escribís el valor nuevo en el mismo renglón. El renglón sigue siendo el mismo, solo cambia su contenido.

Cada vez que un usuario cambia su contraseña, actualiza su dirección o un producto cambia de precio, hay un UPDATE trabajando detrás de escena.

Sintaxis básica

La estructura de UPDATE siempre sigue el mismo patrón: dices qué tabla, qué cambias, y con qué condición.

UPDATE
¿Qué tabla modificas? El nombre de la tabla con los registros a cambiar.
SET
¿Qué columnas cambias y a qué valor? Columna = nuevo valor, separadas por comas si son varias.
WHERE
¿Qué filas se ven afectadas? La condición que deben cumplir las filas a actualizar. ¡Nunca lo omitas!
Subir el salario de Ana Torres
UPDATE empleados SET salario = 3100 WHERE id = 1;

💡
Usa siempre el id en WHERE El id es único por fila. Usarlo en WHERE garantiza que solo se modifica exactamente el registro que querés, sin riesgo de afectar otros con el mismo nombre.

Actualizar varias columnas a la vez

Con SET podés cambiar varias columnas en una sola sentencia separándolas con comas. Es más eficiente que ejecutar un UPDATE por cada columna.

🏠
Analogía del formulario de actualización Cuando actualizás tus datos en un banco — nueva dirección, nuevo teléfono, nuevo email — lo hacés en un solo formulario, no en tres visitas distintas. UPDATE con múltiples columnas en SET funciona igual.
Pedro cambia de ciudad y departamento
UPDATE empleados SET ciudad = 'Quito', departamento = 'Ventas', salario = 1850 WHERE id = 4;

El peligro de olvidar WHERE

Este es el error más costoso en SQL. Sin WHERE, el UPDATE afecta absolutamente todas las filas de la tabla.

✗ Sin WHERE — catastrófico
UPDATE empleados SET salario = 999; -- ¡Todos los salarios -- ahora son 999!
✓ Con WHERE — seguro
UPDATE empleados SET salario = 999 WHERE id = 2; -- Solo Luis Mora -- cambia a 999
⚠️
Regla de oro: nunca hagas UPDATE sin WHERE en producción Un UPDATE sin WHERE en una tabla con miles de registros puede destruir datos en milisegundos. No hay "Ctrl+Z". Siempre verificá con un SELECT primero.
🔒
Analogía del freno de mano El WHERE en un UPDATE es como el freno de mano de un auto estacionado en una pendiente. Podés olvidarlo una vez… pero las consecuencias son irreversibles. Hacelo un hábito automático.

Operadores matemáticos en SET

No siempre sabés el valor nuevo exacto. Podés usar el valor actual de la columna como base para calcular el nuevo valor.

Aumentar el salario de Ingeniería un 10%
UPDATE empleados SET salario = salario * 1.10 WHERE departamento = 'Ingeniería';

Actualizar múltiples filas a la vez

WHERE no solo puede apuntar a una fila. Podés actualizar todas las filas que cumplan una condición con un solo UPDATE.

📢
Analogía del anuncio masivo Es como mandar un correo a todos los empleados del departamento de Ventas con un nuevo código de acceso. No vas uno por uno: mandás una sola comunicación que llega a todos los que cumplen el criterio.
Desactivar todos los empleados de RRHH
UPDATE empleados SET activo = 0 WHERE departamento = 'RRHH';

Combinar condiciones con AND / OR

Subir salario a empleados activos en Quito
UPDATE empleados SET salario = salario + 300 WHERE ciudad = 'Quito' AND activo = 1;
Usar IN para múltiples valores
UPDATE empleados SET departamento = 'Comercial' WHERE id IN (2, 5); -- Actualiza Luis Mora y Sofía León

El proceso seguro para hacer un UPDATE

Antes de ejecutar cualquier UPDATE en una base de datos real, seguí siempre estos pasos:

1

Verificá con SELECT primero

Ejecutá un SELECT con el mismo WHERE para ver exactamente qué filas se van a modificar.

2

Revisá los resultados del SELECT

Confirmá que las filas son las correctas. Si hay más o menos de lo esperado, revisá la condición.

3

Ejecutá el UPDATE

Solo cuando estés seguro de qué filas serán afectadas, ejecutá el UPDATE.

4

Verificá el resultado

Volvé a hacer un SELECT para confirmar que los datos cambiaron correctamente.

Flujo seguro completo
-- Paso 1: verificar qué filas se van a tocar SELECT * FROM empleados WHERE departamento = 'Ventas'; -- Paso 2: si el resultado es correcto, ejecutar UPDATE empleados SET salario = salario + 200 WHERE departamento = 'Ventas'; -- Paso 3: confirmar el resultado SELECT nombre, salario FROM empleados WHERE departamento = 'Ventas';
🛡️
Tip extra: usá transacciones en producción Con START TRANSACTION podés ejecutar el UPDATE, verificar el resultado y decidir si hacés COMMIT (confirmar) o ROLLBACK (deshacer). Es el seguro de vida del UPDATE.
UPDATE con transacción (máxima seguridad)
START TRANSACTION; UPDATE empleados SET salario = salario + 500 WHERE id = 3; -- Verificar antes de confirmar SELECT nombre, salario FROM empleados WHERE id = 3; -- Si todo está bien: COMMIT; -- Si algo salió mal: -- ROLLBACK;

Errores comunes

1. WHERE con columna de texto en vez de id

✗ Arriesgado
UPDATE empleados SET salario = 3000 WHERE nombre = 'Ana'; -- ¿Hay más de un Ana?
✓ Seguro
UPDATE empleados SET salario = 3000 WHERE id = 1; -- id es único siempre

2. Actualizar con el tipo de dato equivocado

✗ Error de tipo
UPDATE empleados SET salario = 'dos mil' WHERE id = 1; -- salario es numérico, -- no acepta texto
✓ Tipo correcto
UPDATE empleados SET salario = 2000 WHERE id = 1; -- número sin comillas ✓

3. Olvidar comas entre columnas en SET

✗ Sin coma
UPDATE empleados SET ciudad = 'Quito' salario = 2000 WHERE id = 1; -- Error de sintaxis
✓ Con coma
UPDATE empleados SET ciudad = 'Quito', salario = 2000 WHERE id = 1; -- coma entre columnas ✓

Resumen: UPDATE completo

Todo junto en un ejemplo real
-- 1. Verificar primero SELECT nombre, salario, ciudad FROM empleados WHERE departamento = 'Ingeniería' AND activo = 1; -- 2. Actualizar con seguridad UPDATE empleados SET salario = salario * 1.15, ciudad = 'Quito' WHERE departamento = 'Ingeniería' AND activo = 1; -- 3. Confirmar cambios SELECT nombre, salario, ciudad FROM empleados WHERE departamento = 'Ingeniería';

Lo que aprendiste

  • UPDATE modifica filas existentes sin borrarlas ni crear nuevas
  • SET define qué columnas cambian y cuál es el nuevo valor
  • WHERE determina qué filas se afectan — siempre es obligatorio en producción
  • Podés actualizar varias columnas en un SET separándolas con comas
  • Podés usar el valor actual en SET: salario = salario + 200
  • Sin WHERE el UPDATE afecta todas las filas de la tabla
  • Verificá siempre con SELECT antes de ejecutar el UPDATE
  • Usá START TRANSACTION / COMMIT / ROLLBACK para máxima seguridad
