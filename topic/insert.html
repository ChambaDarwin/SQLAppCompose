Dominando el INSERT
en MySQL

Antes de consultar datos con SELECT, alguien tuvo que agregarlos. INSERT es el comando que convierte tu base de datos vacía en algo útil.

¿Qué es INSERT?

INSERT INTO es el comando que usas para agregar nuevas filas a una tabla. A diferencia de SELECT (que solo lee), INSERT escribe datos permanentemente en la base de datos.

📝
Analogía del cuaderno de registro: Imagina una recepcionista con un cuaderno donde anota a cada visitante: nombre, hora de llegada, motivo. Cada vez que llega alguien nuevo, ella abre el cuaderno y escribe una fila nueva. INSERT hace exactamente eso: agrega una fila nueva al "cuaderno" (la tabla) con los datos que tú le indicas.

Cada vez que un usuario se registra en una app, compra un producto o envía un formulario, hay un INSERT trabajando detrás de escena.

La tabla que usaremos

Trabajaremos con una tabla llamada productos. Primero veamos cómo fue creada, para entender qué columnas tiene y qué tipo de datos acepta:

Estructura de la tabla productos
CREATE TABLE productos ( id INT AUTO_INCREMENT PRIMARY KEY, nombre VARCHAR(100) NOT NULL, categoria VARCHAR(50), precio DECIMAL(8,2) NOT NULL, stock INT DEFAULT 0, activo TINYINT DEFAULT 1 );

Actualmente la tabla está vacía. A lo largo de esta guía iremos llenándola con INSERT.

ColumnaTipoObligatoriaNotas
idINT—Se genera automáticamente (AUTO_INCREMENT)
nombreVARCHAR(100)✓ SíNo puede quedar vacío (NOT NULL)
categoriaVARCHAR(50)NoPuede quedar NULL
precioDECIMAL(8,2)✓ SíAcepta decimales: 19.99
stockINTNoPor defecto vale 0
activoTINYINTNoPor defecto vale 1 (activo)

Sintaxis básica

La estructura de INSERT siempre sigue el mismo patrón: dices en qué tabla, qué columnas vas a llenar, y qué valores pones en cada una.

INSERT INTO
¿En qué tabla insertas? El nombre de la tabla donde irá la nueva fila.
(col1, col2…)
¿Qué columnas vas a llenar? Lista entre paréntesis, separadas por comas.
VALUES
¿Qué valores tiene cada columna? En el mismo orden que las columnas declaradas.
(v1, v2…)
Los valores concretos. Textos entre comillas simples, números sin comillas.
Insertar un producto completo
INSERT INTO productos (nombre, categoria, precio, stock, activo) VALUES ('Teclado mecánico', 'Electrónica', 89.99, 15, 1);
Estado de la tabla después del INSERT →
idnombrecategoriapreciostockactivo
1 Teclado mecánico Electrónica 89.99 15 1
El id = 1 apareció solo. Como la columna id tiene AUTO_INCREMENT, MySQL la llena automáticamente. No necesitas incluirla en el INSERT.

¿Cómo escribir cada tipo de dato?

El formato de los valores varía según el tipo de dato de la columna. Mezclarlos es uno de los errores más comunes en INSERT.

Tipo de datoCómo escribirloEjemplo
Texto (VARCHAR, TEXT)Entre comillas simples '...''Laptop'
Número entero (INT)Sin comillas42
Número decimal (DECIMAL)Sin comillas, punto como separador19.99
Fecha (DATE)Entre comillas, formato AAAA-MM-DD'2024-03-15'
Fecha y hora (DATETIME)Entre comillas, formato completo'2024-03-15 10:30:00'
Booleano / TINYINT1 para verdadero, 0 para falso1 o 0
Valor nuloLa palabra NULL sin comillasNULL
✉️
Analogía del sobre: Piensa en los tipos de dato como sobres distintos. Un número va en sobre de papel normal (sin comillas). Un texto va en sobre de plástico transparente (comillas simples) para que MySQL sepa que es un contenido literal, no una instrucción.

Insertar múltiples filas a la vez

En vez de escribir un INSERT por cada fila, puedes insertar varias de una sola vez. Es más rápido y más eficiente, especialmente con grandes volúmenes de datos.

🧺
Analogía del supermercado: Ir al supermercado 10 veces a comprar un producto cada vez es ineficiente. Es mucho mejor ir una sola vez con una lista de 10 productos. INSERT múltiple hace lo mismo: un solo viaje a la base de datos con todos los datos.
Insertar 4 productos de una sola vez
INSERT INTO productos (nombre, categoria, precio, stock) VALUES ('Mouse inalámbrico', 'Electrónica', 24.99, 40), ('Monitor 24"', 'Electrónica', 199.00, 8), ('Silla ergonómica', 'Mobiliario', 320.00, 5), ('Libreta de notas', 'Papelería', 3.50, 200);
Estado de la tabla después →
idnombrecategoriapreciostockactivo

Las filas nuevas están resaltadas en verde. Nota que activo se llenó solo con 1 porque definimos DEFAULT 1 al crear la tabla.

Insertar sin llenar todas las columnas

No es obligatorio llenar todas las columnas. Puedes omitir las que tienen un valor por defecto (DEFAULT) o que permiten NULL. MySQL las llenará automáticamente.

Solo nombre y precio (lo mínimo obligatorio)
INSERT INTO productos (nombre, precio) VALUES ('Bolígrafo azul', 1.20);
Fila resultante →
idnombrecategoriapreciostockactivo
6 Bolígrafo azul NULL 1.20 0 1

Observa lo que ocurrió con las columnas omitidas:

Columna¿Qué pasó?
idAUTO_INCREMENT → se asignó 6 automáticamente
categoriaNo tiene DEFAULT ni NOT NULL → quedó NULL
stockDEFAULT 0 → se llenó con 0
activoDEFAULT 1 → se llenó con 1
¿Qué pasa si omites una columna NOT NULL sin DEFAULT? MySQL lanzará un error: ERROR 1364: Field 'nombre' doesn't have a default value. Siempre debes proveer valores para columnas NOT NULL sin DEFAULT.

AUTO_INCREMENT: el ID automático

La columna id con AUTO_INCREMENT es especial. MySQL le asigna automáticamente el siguiente número disponible cada vez que haces un INSERT.

🎟️
Analogía del talonario de turnos: En un banco, cuando llegas te dan un número de turno: 1, 2, 3… Nadie decide el número manualmente, simplemente se asigna el siguiente disponible. AUTO_INCREMENT funciona igual: cada nuevo INSERT recibe el número siguiente automáticamente.
Nunca incluyas el id en el INSERT
-- ✓ Correcto: MySQL asigna el id solo INSERT INTO productos (nombre, precio) VALUES ('Agenda 2024', 8.50); -- Puedes ver el id generado con: SELECT LAST_INSERT_ID();
LAST_INSERT_ID() devuelve el id de la última fila insertada en tu sesión. Muy útil cuando necesitas ese id para insertar en otra tabla relacionada.

INSERT + SELECT: copiar datos entre tablas

Puedes combinar INSERT con SELECT para insertar datos que ya existen en otra tabla (o en la misma). En lugar de VALUES, usas una consulta SELECT.

📋
Analogía de la fotocopiadora: En vez de volver a escribir a mano un documento que ya tienes, lo fotocopias. INSERT + SELECT es la "fotocopiadora" de datos: toma registros de una tabla y los copia a otra sin tener que escribirlos uno a uno.

Supongamos que tenemos una tabla productos_archivo y queremos copiar todos los productos inactivos:

Copiar productos inactivos a otra tabla
INSERT INTO productos_archivo (nombre, precio, categoria) SELECT nombre, precio, categoria FROM productos WHERE activo = 0;
No uses VALUES cuando combinas con SELECT. El SELECT reemplaza al VALUES. Las columnas del SELECT deben coincidir en orden y tipo con las columnas del INSERT.

ON DUPLICATE KEY UPDATE

¿Qué pasa si intentas insertar una fila con un id que ya existe? Normalmente MySQL lanza un error. Con ON DUPLICATE KEY UPDATE le dices: "si ya existe, actualiza estos campos en lugar de fallar".

Insertar o actualizar si ya existe
INSERT INTO productos (id, nombre, precio, stock) VALUES (1, 'Teclado mecánico RGB', 94.99, 20) ON DUPLICATE KEY UPDATE nombre = VALUES(nombre), precio = VALUES(precio), stock = VALUES(stock);
¿Cuándo usarlo? Cuando no sabes si un registro ya existe y quieres "insertar o actualizar" en una sola operación. Este patrón se llama upsert y es muy común en sincronizaciones de datos.

Errores comunes y cómo evitarlos

1. Número de columnas ≠ número de valores

✗ Error
INSERT INTO productos (nombre, precio, stock) VALUES ('Laptop', 999.00); -- ¡Faltan valores para stock!
✓ Correcto
INSERT INTO productos (nombre, precio, stock) VALUES ('Laptop', 999.00, 3); -- 3 columnas, 3 valores ✓

2. Texto sin comillas simples

✗ Error
INSERT INTO productos (nombre, precio) VALUES (Laptop, 999.00); -- MySQL interpreta Laptop -- como nombre de columna
✓ Correcto
INSERT INTO productos (nombre, precio) VALUES ('Laptop', 999.00); -- Texto siempre entre -- comillas simples ✓

3. Insertar en columna NOT NULL con valor NULL

✗ Error
INSERT INTO productos (nombre, precio) VALUES (NULL, 999.00); -- nombre es NOT NULL, -- no puede ser NULL
✓ Correcto
INSERT INTO productos (nombre, precio) VALUES ('Laptop', 999.00); -- nombre tiene valor, -- todo bien ✓
Tip de seguridad: Si tu texto puede contener comillas simples (ejemplo: O'Brien), escápalo con una barra invertida: 'O\'Brien' o usa doble comilla simple: 'O''Brien'. Nunca construyas un INSERT concatenando texto del usuario directamente — usa consultas preparadas para evitar SQL Injection.

Resumen: INSERT completo

Un INSERT que combina todo lo visto en esta guía:

Insertar múltiples productos con todos los campos
-- Insertar varios productos a la vez INSERT INTO productos (nombre, categoria, precio, stock, activo) VALUES ('Webcam HD', 'Electrónica', 49.99, 25, 1), ('Auriculares BT', 'Electrónica', 65.00, 18, 1), ('Lámpara escritorio','Mobiliario', 22.50, 30, 1); -- Ver el id del último insertado SELECT LAST_INSERT_ID(); -- Verificar los datos insertados SELECT * FROM productos ORDER BY id DESC LIMIT 3;

Lo que aprendiste

  • INSERT INTO agrega nuevas filas a una tabla
  • El orden de los valores debe coincidir con el orden de las columnas
  • Textos entre comillas simples, números sin comillas, fechas en formato AAAA-MM-DD
  • Puedes insertar múltiples filas en un solo INSERT separando los VALUES con comas
  • Columnas con DEFAULT o que aceptan NULL se pueden omitir
  • AUTO_INCREMENT asigna el id automáticamente — nunca lo incluyas en el INSERT
  • INSERT + SELECT copia datos desde otra tabla sin escribir VALUES
  • ON DUPLICATE KEY UPDATE permite "insertar o actualizar" en una operación
