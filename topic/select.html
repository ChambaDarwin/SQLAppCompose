Dominando el SELECT
en MySQL

Todo en SQL empieza con una pregunta. SELECT es la forma de hacerle preguntas a tu base de datos y obtener exactamente lo que necesitas.

¿Qué es SELECT?

SELECT es el comando que usas para leer o consultar datos de una base de datos. No modifica nada, solo recupera información. Es el comando más usado en SQL y el punto de partida para entender todo lo demás.

📋
Analogía del mesero: Imagina que la base de datos es un restaurante con muchas mesas llenas de platos. SELECT es como pedirle al mesero: "Tráeme todos los platos que tengan pollo y que estén en la mesa 3, ordenados por precio." El mesero va, busca, y te trae exactamente lo que pediste. La cocina (los datos) no cambia en absoluto.

En términos simples: SELECT le dice a MySQL qué quieres ver, de qué tabla, con qué condiciones.

Sintaxis básica

La estructura mínima de un SELECT es muy simple. Solo necesitas decirle qué columnas y de qué tabla:

SELECT
¿Qué columnas quieres ver? Escribe los nombres separados por comas, o usa * para todas.
FROM
¿De qué tabla? El nombre de la tabla donde están los datos.
WHERE
¿Con qué condición? (opcional) Filtra las filas que cumplen la condición.
ORDER BY
¿En qué orden? (opcional) Ordena los resultados.
LIMIT
¿Cuántos resultados? (opcional) Limita la cantidad de filas devueltas.
Traer todos los empleados
SELECT * FROM empleados;
Resultado →
idnombreciudaddepartamentosalarioactivo
1Ana TorresQuitoIngeniería28001
2Luis MoraGuayaquilVentas19001
3Carmen RíosQuitoIngeniería32001
4Pedro VegaCuencaRRHH16000
5Sofía LeónGuayaquilVentas21001
6Marco DíazQuitoRRHH17501
El asterisco * significa "todas las columnas". Es útil para explorar una tabla, pero en proyectos reales es mejor especificar solo las columnas que necesitas.

Elegir columnas específicas

En vez de traer todo, puedes pedir solo las columnas que te interesan. Es más eficiente y más claro.

📰
Analogía del periódico: Cuando lees el periódico, no lees todo: solo abres la sección de deportes. SELECT con columnas específicas es lo mismo: vas directo a lo que te importa, ignoras el resto.
Solo nombre y salario
SELECT nombre, salario FROM empleados;
Resultado →
nombresalario
Ana Torres2800
Luis Mora1900
Carmen Ríos3200
Pedro Vega1600
Sofía León2100

Filtrar con WHERE

WHERE es el filtro. Le dices a MySQL: "solo devuélveme las filas que cumplen esta condición". Sin WHERE, obtienes todo; con WHERE, obtienes exactamente lo que buscas.

🔍
Analogía del buscador: WHERE es como escribir algo en Google. Sin escribir nada, te muestra millones de páginas. Cuando escribes "empleados en Quito", Google filtra y te muestra solo lo relevante. WHERE hace lo mismo con tus filas.
Empleados de Quito
SELECT nombre, ciudad, salario FROM empleados WHERE ciudad = 'Quito';
Resultado →
nombreciudadsalario
Ana TorresQuito2800
Carmen RíosQuito3200
Marco DíazQuito1750

Combinar condiciones con AND y OR

Puedes combinar varias condiciones. AND exige que se cumplan ambas. OR acepta que se cumpla al menos una.

Ingenieros en Quito
SELECT nombre, departamento, ciudad FROM empleados WHERE ciudad = 'Quito' AND departamento = 'Ingeniería';
Resultado →
nombredepartamentociudad
Ana TorresIngenieríaQuito
Carmen RíosIngenieríaQuito
Quito O Cuenca
SELECT nombre, ciudad FROM empleados WHERE ciudad = 'Quito' OR ciudad = 'Cuenca';

Ciudades específicas con IN
SELECT nombre, ciudad FROM empleados WHERE ciudad IN ('Guayaquil', 'Cuenca');
🛒
Analogía de IN: Es como decirle al mesero "quiero cualquier cosa del menú que sea pizza, pasta o ensalada". No enumeras condiciones una a una, sino que le das una lista de opciones aceptables.

Buscar texto con LIKE

LIKE sirve para buscar texto parcial. Usa % como comodín que representa cualquier cantidad de caracteres.

🔎
Analogía del buscador de contactos: Cuando en tu celular buscas "Mar" en los contactos, aparece "María", "Marco", "Mariana". LIKE hace lo mismo: LIKE 'Mar%' encuentra todo lo que empieza con "Mar".
PatrónEncuentra
'Mar%'Empieza con "Mar" → Marco, María, Mariana
'%ez'Termina en "ez" → Gómez, López, Pérez
'%or%'Contiene "or" → Torres, Mora, Flor
Nombres que contienen "or"
SELECT nombre FROM empleados WHERE nombre LIKE '%or%';
Resultado →
nombre
Ana Torres
Luis Mora

Ordenar con ORDER BY

ORDER BY ordena los resultados. ASC (ascendente, de menor a mayor) es el valor por defecto. DESC invierte el orden.

📊
Analogía de la lista de canciones: Imagina que tienes una lista de reproducción. Puedes ordenar las canciones por duración, de la más corta a la más larga (ASC) o al revés (DESC). ORDER BY hace lo mismo con tus datos.
Empleados ordenados por salario (mayor a menor)
SELECT nombre, salario FROM empleados ORDER BY salario DESC;
Resultado →
nombresalario
Carmen Ríos3200
Ana Torres2800
Sofía León2100
Luis Mora1900
Marco Díaz1750
Pedro Vega1600

Puedes ordenar por varias columnas. La segunda columna desempata cuando hay valores iguales:

Ordenar por departamento y luego por nombre
SELECT nombre, departamento FROM empleados ORDER BY departamento ASC, nombre ASC;

Limitar resultados con LIMIT

LIMIT restringe cuántas filas devuelve la consulta. Es muy útil cuando solo necesitas los primeros N resultados o para paginar datos.

Los 3 empleados con mayor salario
SELECT nombre, salario FROM empleados ORDER BY salario DESC LIMIT 3;
Resultado →
nombresalario
Carmen Ríos3200
Ana Torres2800
Sofía León2100
LIMIT con OFFSET: Puedes combinar LIMIT con OFFSET para paginar. LIMIT 3 OFFSET 3 salta los primeros 3 y trae los siguientes 3.

Renombrar con AS

AS crea un alias, es decir, un nombre temporal para una columna en el resultado. Hace los resultados más legibles.

Columnas con alias descriptivos
SELECT nombre AS 'Empleado', salario AS 'Sueldo mensual', ciudad AS 'Ubicación' FROM empleados;
Resultado →
EmpleadoSueldo mensualUbicación
Ana Torres2800Quito
Luis Mora1900Guayaquil
Carmen Ríos3200Quito

Valores únicos con DISTINCT

DISTINCT elimina los duplicados del resultado. Si hay varias filas con el mismo valor, solo aparece una.

🎯
Analogía del casillero: Imagina una pila de formularios donde cada persona puso su ciudad. Muchos repiten "Quito". DISTINCT es como revisar la pila y quedarte solo con una copia de cada ciudad, sin repetidos.
¿Qué ciudades existen en la tabla?
SELECT DISTINCT ciudad FROM empleados;
Resultado → (sin "Quito" repetido)
ciudad
Quito
Guayaquil
Cuenca

Resumen: SELECT completo

Aquí una consulta que combina todo lo que aprendiste:

Consulta completa con todos los elementos
-- Empleados activos de Ingeniería en Quito, -- ordenados por salario, solo los 2 primeros SELECT nombre AS 'Empleado', salario AS 'Sueldo', departamento FROM empleados WHERE activo = 1 AND ciudad = 'Quito' AND departamento = 'Ingeniería' ORDER BY salario DESC LIMIT 2;
Resultado →
EmpleadoSueldodepartamento
Carmen Ríos3200Ingeniería
Ana Torres2800Ingeniería

Lo que aprendiste

  • SELECT * trae todas las columnas, SELECT col1, col2 solo las que quieres
  • WHERE filtra filas con condiciones usando =, >, <, BETWEEN, IN, LIKE
  • AND y OR combinan múltiples condiciones
  • ORDER BY ASC/DESC ordena los resultados
  • LIMIT restringe cuántas filas se devuelven
  • AS renombra columnas para hacer los resultados más legibles
  • DISTINCT elimina valores duplicados
Recuerda: SELECT nunca modifica tus datos. Solo los lee. Es completamente seguro experimentar con SELECT sin miedo a romper nada en tu base de datos.
